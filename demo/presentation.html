The Problem

Manual Bug Triage is Broken

  • Large volume of aged, untriaged bugs in CSFMD project
  • Manual triage is slow, repetitive, and error-prone
  • Developers spend 30%+ time on triage instead of building
  • Bugs age for weeks before reaching the right developer

No Linkage Between

  • Bug → which area of code is affected?
  • Code → who owns this module?
  • Owner → what's the fix priority?
  • Fix → did it pass? Should we close the bug?

Result: Bugs pile up, customers wait, quality degrades.

pipeline.py — The Orchestrator

What It Does

  • Main entry point — runs the 6-step pipeline
  • Fetches → Deduplicates → Prioritizes → Analyzes → Pushes → Updates JIRA
  • Supports --dry-run, single-bug mode, and limit options
  • Logs every run to pipeline_log.json
# Usage examples python pipeline.py # Full run python pipeline.py --dry-run # Analyze only python pipeline.py --bug CSFMD-10288 # Single bug python pipeline.py --limit 10 # Top 10 bugs # 6-Step Pipeline: [1/6] Fetch NEW bugs from JIRA [2/6] Deduplicate against tracked bugs [3/6] Prioritize (Highest → Low) [4/6] Analyze against TFS source [5/6] Push fixes & raise PRs [6/6] Update JIRA statuses

jira_client.py — JIRA Integration

Capabilities

  • Authenticates via JIRA Cloud REST API (API token)
  • Fetches bugs from CSFMD project with JQL filters
  • Parses ADF (Atlassian Document Format) → plain text
  • Deduplicates against already-analyzed bugs
  • Transitions issues (New → In Development → Code Review)
  • Adds agent comments with root cause & PR links
  • Priority sorting: Highest → High → Medium → Low → Lowest
# Key functions: fetch_new_bugs(status="New") → JQL query → CSFMD project bugs deduplicate_bugs(bugs, existing) → Filter out already-tracked transition_issue(key, "In Development") → Move bug through JIRA workflow add_comment(key, message) → Post agent analysis as comment priority_sort_key(bug) → Highest=0, High=1, ... Lowest=4

tfs_client.py — TFS Source Code Access

How It Works

  • Connects to TFS REST API via NTLM auth (Windows creds)
  • Browses $/OnBase/DEV source tree
  • Maps bug keywords → source paths automatically
  • Caches fetched files locally in tfs_cache/

Source Path Mapping

  • "DataValidation" → Hyland.Web.Resources.Scripts
  • "keyword panel" → Hyland.Controls.Web/KeywordPanel
  • "web client" → Hyland.Controls.Web
  • "admin" → Hyland.Web.Administration

bug_analyzer.py — The Brain

Classification & Root Cause

  • Classifies bugs into codebases: Web Client, Unity, EDM, Core, Workflow
  • Pattern-matches bug text against known issue signatures
  • Detects: DataValidation gaps, cascading dataset bugs, keyword panel errors, MIKG/SIKG issues, locale mismatches
  • Marks bugs as: FIX_PENDING, NEEDS_MANUAL_REVIEW, or FLAGGED_INVALID
# Root cause detection patterns: DataValidation + "invalid" → Missing JS file for OS version → Fixable by agent ✓ Cascading + "hidden" → Hidden parent keyword → empty child → Needs manual review Keyword panel + "collapse" → Null response from server → Needs manual review MIKG/SIKG + "expand" → Instance handling issue → Needs manual review intermittent | NVDA → Flagged as invalid / non-code bug

github_push.py — Auto PR Creation

End-to-End Git Automation

  • Ensures local repo is configured & up to date
  • Creates feature branches: feature/agent-tests-{IDs}-{date}
  • Copies fix files into correct project directory
  • Commits with descriptive messages linked to JIRA IDs
  • Pushes to GitHub and creates Pull Requests
  • PR body includes: summary, root cause, fix description, affected files
# Auto-generated PR example: Branch: feature/tamisra.sarkar/ agent-tests-10288-20260415 PR Title: fix(CSFMD-10288): DataValidation resorting to default locale PR Body: ## Bug Fix: CSFMD-10288 **Root Cause:** DataValidation JS file missing for detected OS version **Fix:** Add DataValidation2025.js and register in EmbeddedScripts.cs **Affected Files:** - EmbeddedScripts.cs *Generated by CSFMD Bug Agent*
